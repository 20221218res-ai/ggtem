# GGtem 운영 오픈 매뉴얼

최종 업데이트: 2026-05-17

실제 유저 오픈 전후에 운영자가 바로 보는 짧은 매뉴얼이다.

## 서비스 구조

- 유저 도메인: `ggtem.com`
- 어드민 도메인: `topofword.com`
- 호스팅: Vercel
- DB: Supabase PostgreSQL
- 충전/출금: 관리자 수동 승인
- 고위험 영역: 지갑 잔액, 에스크로, 정산, 출금, 관리자 권한, 수수료 정책

## 현재 보안 강화 상태

확인된 항목:

- 유저/어드민 도메인 분리 문서화: `ggtem.com`, `topofword.com`
- 관리자 로그인은 비밀번호 확인 뒤 이메일 OTP를 요구
- 관리자 OTP는 만료, 재발송 쿨다운, 실패 횟수, 잠금 처리가 있음
- 주요 관리자 작업은 비밀번호 재확인을 요구하는 흐름으로 정리됨
- `/admin/deposit-addresses`는 `SUPER` 전용이어야 함
- 충전 승인은 TXID가 필요하고 중복 TXID는 거부되어야 함
- 출금 완료는 외부 송금 TXID가 필요함
- 출금 신청은 결제 PIN을 요구함
- 결제 PIN은 반복 실패 시 잠금 정책이 문서화되어 있음
- TRC20/BEP20 출금 주소 형식 검증이 있음
- 관리자 감사 로그와 정산 원장 화면이 운영 확인 지점으로 있음

오픈 전 남은 점검:

- Vercel 프로덕션 환경변수를 대시보드에서 직접 확인
- Supabase Security Advisor 실행 및 치명 경고 해소
- Vercel 팀/계정 2FA 활성화 확인
- 프로덕션 관리자 계정이 실제 운영자만 남았는지 확인
- 데모/테스트 관리자 계정 비활성화 또는 삭제
- 회원가입, 비밀번호 재설정, 이메일 인증, 관리자 OTP 메일 발송 확인
- Telegram 관리자 알림을 쓰는 경우 실제 수신 확인
- 공개 업로드가 늘기 전 내구성 있는 스토리지 전환 여부 결정

## 일일 운영 체크리스트

업무 시작:

- `https://topofword.com/admin` 접속
- `/admin/deposits` 대기 충전 확인
- `/admin/withdrawals` 또는 `/admin/finance` 대기 출금 확인
- `/admin/disputes`, `/admin/order-chats` 긴급 거래 이슈 확인
- `/admin/risk` 의심 유저, 매물, 출금 패턴 확인
- `/admin/audit` 예상 밖 관리자 작업 확인
- `/admin/finance/ledger` 비정상 지갑/플랫폼 수익 원장 확인

처리 중:

- TXID 없이 충전 승인 금지
- 외부 송금 TXID 없이 출금 완료 금지
- 출금 완료 전 체인, 주소, 금액, 수수료, 유저를 재확인
- 분쟁 처리 전 구매자 환불/판매자 지급 사유 기록
- 계정 거래는 계정 정보 인계 확인 후 완료
- 승인된 관리자 흐름 밖에서 지갑 잔액 직접 수정 금지

업무 종료:

- `/admin/finance/reconciliation`에서 마감 리포트 저장
- 실제 돈이 이동한 날은 원장/정산 CSV 내보내기
- 주문 수수료, 출금 수수료, 프리미엄 수수료 원장 확인
- 실패한 로그인이나 의심스러운 관리자 접근 기록 확인

## 돈 흐름 원칙

충전:

- 유저가 충전 요청 생성
- 관리자가 외부 입금 확인
- 관리자가 TXID와 함께 승인
- 승인 후에만 유저 지갑 잔액 증가

주문/에스크로:

- 주문 생성 시 구매자 금액은 에스크로로 잠김
- 주문 완료 또는 판매자 지급 분쟁 처리 후에만 판매자 정산
- 플랫폼 매출은 주문 생성 시점이 아니라 완료/판매자 지급 시점에 인식

출금:

- 유저는 출금 정책과 결제 PIN 검증을 통과해야 함
- 관리자는 주소, 금액, 수수료, 위험 플래그를 확인
- 실제 외부 송금 후에만 완료 처리
- 완료 처리에는 외부 송금 TXID가 필요

## 수수료 정책 상태

- 현재 코드 확인 기준 거래 수수료 계산은 `600` basis points, 즉 6%로 보임
- 사업/운영 목표는 5% 변경 예정
- 5%는 적용 완료가 아니라 예정으로만 표시
- 코드, UI 문구, 관리자 리포트, 정산 내보내기, 스모크 테스트가 함께 맞기 전에는 유저에게 5%로 공지하지 말 것
- 출금 수수료는 현재 코드 기준 최소 20 USDT, 20-100 USDT는 1 USDT, 100 USDT 초과는 0.5 USDT로 확인됨

## PWA 예정 항목

PWA는 아직 완료 항목으로 보지 않는다.

완료 처리 전 확인:

- 웹 앱 manifest 추가 및 검증
- 모바일 설치용 아이콘 추가
- 지갑, 주문, 어드민, 인증 페이지 캐시 정책 결정
- 개인 지갑/어드민/주문/API 응답은 캐시 금지
- 공개 페이지만 오프라인 fallback 적용 검토
- Android Chrome, iOS Safari 설치 동작 확인
- 푸시 알림 범위 결정
- 푸시가 없어도 금융/운영 알림은 서버 기준으로 신뢰 가능해야 함

## 오픈 Go/No-Go

아래가 모두 참일 때만 오픈:

- `npm.cmd run typecheck` 통과
- `npm.cmd run build` 통과
- Prisma 마이그레이션이 프로덕션 Supabase에 적용됨
- `topofword.com`에서 관리자 MFA 메일 동작
- `ggtem.com`에서 회원가입, 로그인, 이메일 인증, 비밀번호 재설정 동작
- 소액으로 충전, 에스크로, 주문 완료, 분쟁, 출금, 정산 마감 테스트 통과
- Vercel 프로덕션 env에 로컬 DB URL 없음
- 데모 계정/도구 비활성화
- Supabase 백업 활성화
- 유저에게 보이는 수수료 정책과 배포된 코드가 일치
